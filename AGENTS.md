# AGENTS.md

See `CLAUDE.md` for the authoritative rules governing this repository (one-way sync, manual pushes, the `app_files` non-null invariant). Those rules apply to all agents.

## Cursor Cloud specific instructions

**This is NOT a dev repo.** It is the public delivery mirror for the GlideOS Design Workbench. There is no application to build, no package manager, no dependencies, no test suite, and no dev server here. The tracked content is static delivery data consumed by installed GlideOS agents:

- `manifest.json` — scaffold manifest; `app_files` lists the files installers fetch. Must never be null/empty (see `CLAUDE.md`).
- `manifest-hashes.json` / `installer-manifest-hashes.json` — SHA-256 pins for delivered files.
- `workbench-install.md` — bootstrap install doc.
- `LICENSE`, `README.md`, `.claude/stack-config.json`.

The source-of-truth files listed in `manifest.json` (e.g. `apps/design-workbench/server.ts`) do **not** live in this repo; do not expect an `apps/` tree here. Never hand-edit `manifest.json` / `workbench-install.md` — they are regenerated by the private repo's `./sync-public.sh`.

### "Running" / validating this repo

Since there is nothing to run, validation means confirming the delivery artifacts are well-formed and consistent. All tools below are preinstalled system binaries (`node`, `python3`, `jq`, `git`) — nothing needs installing, so the startup update script is a no-op.

- Validate JSON: `jq empty manifest.json manifest-hashes.json installer-manifest-hashes.json .claude/stack-config.json`
- Check the critical invariant: `jq -e '(.app_files|length) > 0' manifest.json`
- Consumer integrity (every `app_files` path has a hash): compare `jq -r '.app_files[].path' manifest.json` against `jq -r '.files|keys[]' manifest-hashes.json`.
- End-to-end (what an installed agent actually does): fetch the published manifest with no auth — `curl -fsSL https://raw.githubusercontent.com/bschonbrun/glideos-design-workbench-public/main/manifest.json`.
